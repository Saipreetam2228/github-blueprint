# GitHub Project Boards

GitHub Project Boards are tools used to manage tasks and track project progress.

They work similarly to Kanban boards and help teams organize work visually.

---

## Why Project Boards are Useful

Project Boards help teams:

- track tasks
- manage workflows
- organize development work
- monitor project progress

---

## Board Structure

A typical project board contains columns such as:

```
To Do
In Progress
Done
```

Each task is represented as a **card** that moves across columns.

---

## Creating a Project Board

### Step 1

Open the repository on GitHub.

### Step 2

Click the **Projects** tab.

### Step 3

Click **New Project**.

### Step 4

Choose a template or start with a blank board.

---

## Adding Tasks

Tasks can be added to the board using:

- Issues
- Pull Requests
- Custom notes

Example task:

```
Add documentation for GitHub cloning feature
```

---

## Example Workflow

1. Add tasks to **To Do**
2. Move tasks to **In Progress** while working
3. Move tasks to **Done** when completed

---

## Benefits

Project Boards provide:

- visual project tracking
- better team coordination
- improved productivity
