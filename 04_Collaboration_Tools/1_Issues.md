# Issues in GitHub

Issues are used to track tasks, bugs, and feature requests in a GitHub repository.

They help teams organize work and discuss improvements.

---

## Why Issues are Important

Issues help developers:

- report bugs
- request new features
- track tasks
- discuss improvements

They act as a **project task management system**.

---

## Creating an Issue

### Step 1

Open the repository on GitHub.

### Step 2

Click the **Issues** tab.

### Step 3

Click **New Issue**.

### Step 4

Add a title and description.

Example:

```
Bug: Login button not working
```

### Step 5

Click **Submit new issue**.

---

## Issue Features

GitHub Issues support several useful features:

- Labels
- Assignees
- Milestones
- Comments

---

## Labels

Labels help categorize issues.

Examples:

```
bug
enhancement
documentation
help wanted
```

---

## Assignees

Issues can be assigned to specific developers responsible for solving the problem.

---

## Milestones

Milestones group issues together for a specific goal or project phase.

Example:

```
Version 1.0 Release
```
