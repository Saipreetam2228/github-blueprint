# GitHub Stars and Forks

GitHub provides features that allow developers to interact with repositories.  
Two of the most common features are **Stars** and **Forks**.

These features help measure the popularity and usefulness of a project.

---

## GitHub Stars

A star is similar to a bookmark.

Developers star a repository to:

- show appreciation for the project
- bookmark useful repositories
- follow project updates

### Example

A developer finds a useful open-source project and stars the repository to easily find it later.

---

## GitHub Forks

Forking means creating a personal copy of someone else's repository.

Developers fork repositories to:

- experiment with the code
- contribute to open-source projects
- modify the project independently

### Example

A developer forks a project, adds improvements, and then creates a Pull Request to contribute those changes.

---

## Stars vs Forks

| Feature | Stars | Forks |
|------|------|------|
| Purpose | Bookmark repository | Copy repository |
| Usage | Show interest | Modify or contribute |
| Code Changes | No | Yes |

---

## Benefits

Stars and forks help developers:

- discover popular projects
- contribute to open source
- track interesting repositories
