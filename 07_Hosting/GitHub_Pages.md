# GitHub Pages

GitHub Pages is a feature that allows developers to host websites directly from a GitHub repository.

It is commonly used for personal portfolios, documentation websites, and project demos.

---

## What GitHub Pages Can Host

GitHub Pages can host:

- portfolio websites
- project documentation
- blogs
- static websites

---

## How GitHub Pages Works

GitHub Pages takes files from a repository and publishes them as a website.

These files are usually written using:

- HTML
- CSS
- JavaScript
- Markdown

---

## Enabling GitHub Pages

To enable GitHub Pages:

1. Open the repository
2. Click **Settings**
3. Go to **Pages**
4. Select a branch (usually `main`)
5. Save the settings

After enabling, GitHub generates a website link.

Example:

```
https://username.github.io/repository-name
```

---

## Benefits of GitHub Pages

GitHub Pages provides:

- free website hosting
- easy deployment
- integration with GitHub repositories
- automatic updates when code changes

---

## Example Use Cases

Developers commonly use GitHub Pages for:

- personal developer portfolios
- open source documentation
- project demonstrations
