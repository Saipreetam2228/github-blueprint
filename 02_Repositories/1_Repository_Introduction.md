# Repository Introduction

A repository (repo) is the place where a project lives on GitHub.

It contains all the files, folders, and history of changes related to a project.

Repositories allow developers to store, manage, and track their code efficiently.

---

## What a Repository Contains

A typical repository may include:

- source code
- documentation
- configuration files
- images and assets
- commit history

---

## Key Features of a Repository

GitHub repositories provide several powerful features:

- Version control
- Branching
- Pull requests
- Issue tracking
- Collaboration tools
- Project management

---

## Public vs Private Repositories

### Public Repository
Anyone on the internet can see the code.

Used for:

- open-source projects
- portfolios
- public documentation

### Private Repository
Only authorized users can access the repository.

Used for:

- company projects
- confidential work
- personal development

---

## Why Repositories Are Important

Repositories help developers:

- organize project files
- track changes over time
- collaborate with teams
- maintain project history
