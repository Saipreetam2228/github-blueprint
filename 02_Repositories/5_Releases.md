# GitHub Releases

GitHub Releases are used to publish stable versions of a project.

A release allows developers to package their project with version numbers and share it with users.

---

## Why Releases are Important

Releases help developers:

- distribute software versions
- track project progress
- share downloadable files
- maintain version history

---

## Example Versioning

Projects often follow version numbers such as:

```
v1.0.0
v1.1.0
v2.0.0
```

This helps users understand which version of the software they are using.

---

## How to Create a Release

### Step 1

Open your repository on GitHub.

### Step 2

Click **Releases** on the right side.

### Step 3

Click **Create a new release**.

### Step 4

Add a **tag version**.

Example:

```
v1.0
```

### Step 5

Add a **release title** and description.

### Step 6

Click **Publish release**.

---

## What Can Be Included in a Release

A release may contain:

- compiled programs
- documentation
- installation files
- project assets

---

## Example

A typing tutor project might release:

```
TypingTutor-v1.0.zip
```

Users can download and use this version.
