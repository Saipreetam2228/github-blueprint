# Forking Repositories

Forking a repository means creating a copy of someone else's repository in your own GitHub account.

This allows you to experiment with changes without affecting the original project.

---

## Why Forking is Important

Forking is commonly used for:

- contributing to open source projects
- experimenting with code
- making improvements to existing projects

---

## How to Fork a Repository

### Step 1

Open the repository you want to fork.

### Step 2

Click the **Fork** button at the top right of the repository page.

### Step 3

GitHub will create a copy of the repository in your account.

---

## What Happens After Forking

After forking:

- You get a full copy of the repository.
- You can make changes freely.
- The original repository remains unchanged.

---

## Contributing After Forking

Typical workflow:

1. Fork the repository
2. Clone your fork
3. Create a new branch
4. Make changes
5. Commit and push changes
6. Create a Pull Request

---

## Example

Original repository:

```
github.com/original-owner/project
```

Your fork:

```
github.com/your-username/project
```
