# Code Review in GitHub

Code review is the process of examining code changes before they are merged into the main project.

It is usually done through Pull Requests on GitHub.

---

## Why Code Review is Important

Code reviews help teams:

- improve code quality
- detect bugs early
- share knowledge between developers
- maintain coding standards

---

## How Code Review Works

Typical workflow:

1. A developer creates a new branch.
2. The developer makes changes and pushes the branch.
3. A Pull Request is created.
4. Other developers review the code.
5. Feedback or suggestions are provided.
6. The Pull Request is approved and merged.

---

## What Reviewers Can Do

Reviewers can:

- comment on specific lines of code
- suggest improvements
- request changes
- approve the Pull Request

---

## Example Review Process

Developer A creates a Pull Request.

Developer B reviews the code and comments:

```
Consider renaming this variable for clarity.
```

Developer A updates the code and pushes the changes.

After approval, the Pull Request is merged.

---

## Benefits of Code Review

Code reviews help teams:

- maintain high-quality code
- reduce errors
- improve collaboration
