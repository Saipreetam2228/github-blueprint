# Pull Requests in GitHub

A Pull Request (PR) is a feature that allows developers to propose changes to a repository.

It is commonly used when working with branches or contributing to open source projects.

---

## Why Pull Requests are Important

Pull Requests allow teams to:

- review code before merging
- discuss changes
- suggest improvements
- maintain code quality

---

## Basic Pull Request Workflow

Typical workflow:

1. Create a branch
2. Make changes to the code
3. Commit and push the changes
4. Create a Pull Request
5. Review and discuss the changes
6. Merge the Pull Request

---

## Creating a Pull Request

### Step 1

Push your branch to GitHub.

Example:

```
git push origin feature-login
```

### Step 2

Open the repository on GitHub.

### Step 3

Click **Compare & Pull Request**.

### Step 4

Add a title and description.

### Step 5

Click **Create Pull Request**.

---

## Code Review

Other developers can:

- comment on code
- request changes
- approve the Pull Request

This helps improve the overall quality of the project.

---

## After Approval

Once the Pull Request is approved, it can be **merged into the main branch**.
